AI 痕迹速查表

让一个界面被一眼看出是机器生成的那些具体特征（tell）。这张卡会随课程增长；下面这十五条来自第 01 课。 来源缩写：DTF = design-taste-frontend/SKILL.md。

Visual & CSS · 视觉与 CSS

glow

AI 紫蓝渐变（AI purple/blue gradient）

紫到蓝的网格渐变加霓虹按钮辉光。最容易被认出的 AI 指纹。 改法：暖中性底色，一个平涂强调色，不发光。

DTF §4.2（the Lila rule）· §9.A
#000000

纯黑（Pure black）

用 #000 做背景或文字会压掉层次，显得是数字默认值。 改法：近黑（zinc-950 / 暖炭黑）。

DTF §9.A · §8.B

Typography · 排版

Aa

到处都是 Inter（Inter everywhere）

那个没有个性的默认字体。不丑，只是隐形，于是每个页面都像套模板。 改法：换一款有声音的展示字体（Cabinet Grotesk、Satoshi……）。

DTF §4.1 · §9.B
—

破折号花活（the em-dash flourish）

模型标志性的标点拐杖。彻底禁用：整页任何地方都不能有。 改法：句号、逗号、冒号或括号。

DTF §9.G

Layout · 布局

centered
H1

暗色网格上的居中英雄区（centered hero over dark mesh）

模型永远最先尝试的默认英雄区。等于说：没有做任何构图选择。 改法：不对称分栏，文字一边、素材一边。

DTF §0.D · §4.3

三张等宽功能卡（three equal feature cards）

图标 + 粗体词 + 一句话，三列并排，带边框带阴影。 改法：不对称列表、之字排布，或有节奏的 bento。

DTF §9.C · §4.7
00 / INDEX

编号眉标（section-number eyebrow）

每个标题上都挂个编号小标签（“00 / How it works”）。套模板的节奏。 改法：每三节最多一个眉标；用大白话命名。

DTF §4.7 · §9.F
$ run
✓ ok
✓ merge

div 拼的假截图（div-based fake screenshot）

用样式化的 div 拼出假终端 / 仪表盘 / 红绿灯窗口。头号痕迹。 改法：用真截图、生成图，或干脆不放。

DTF §4.8 · §9.E

Content & Copy · 内容与文案

Nexus

创业烂大街的品牌名（startup-slop brand name）

Nexus、Acme、Apex、Cloudly：generic 的生造名，常给某个字母上强调色。 改法：一个朴素、能读出口、有描述性的名字。

DTF §9.D
Unleash

废话动词文案（filler-verb copy）

Elevate、Unleash、Seamless、Next-Gen：什么都没说的陈词滥调。 改法：写具体动作（“开始一次评审”）。

DTF §9.D · §4.9
99.99%

假完美统计数字（fake-perfect statistic）

可疑的整数、没有出处的生造指标，读起来像幻觉。 改法：用真实、带零头、有出处的数字，或不放。

DTF §9.D · §4.9
Sarah Chen
Developer

无名氏证言（the Jane Doe testimonial）

通用名字 + 含糊头衔 + 没有公司。零具体信息的署名。 改法：真名、具体职位、真实公司。

DTF §9.D
✦ v2.0 BETA

英雄区里的版本标（version label in the hero）

把“EARLY ACCESS”“INVITE-ONLY PREVIEW”当眉标。是身份表演，不是价值。 改法：换成一句说清产品做什么的眉标。

DTF §9.F
↓ Scroll

下滑提示（scroll cue）

“↓ Scroll to explore.” 用户就站在英雄区，他们知道怎么滚。 改法：删掉。让内容自己把视线往下带。

DTF §9.F
● Live

装饰性状态点（decorative status dots）

导航项前的彩色 ●，或假的“Live”徽章，只是装饰而非真实状态。 改法：删掉，或让这个点真的代表某种状态。

DTF §9.F
