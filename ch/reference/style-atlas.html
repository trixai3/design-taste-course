风格图鉴 · The Style Atlas

可以拿去指挥 AI 的、有名字的风格家族。读一个家族只问三个问题：substrate（基底）、hierarchy（层级载体）、edge signature（边缘签名）。第 04-05 课覆盖了十个家族，下面再加上第 06 课那两种下定决心的 Brutalist 模式。

亮色家族（第 04 课）

01 · Premium Utilitarian Minimalism

闻起来像：Notion、Craft、Linear docs。一份伪装成应用的文档。

基底
纯白 #FFFFFF 或暖骨白 #F7F6F3
层级
编辑感衬线大标题（Lyon、Newsreader）配干净无衬线（SF Pro、Geist、Switzer）
边缘
精确 1px solid #EAEAEA；圆角 4-12px；阴影近乎为零（< 0.05 不透明度）
招牌
伪 OS 窗口外壳、<kbd> 按键、洗淡的粉彩标签、墨色文字 #2F3437、灰 #787774
何时用
生产力工具、文档型产品、需要读起来可信而安静的 SaaS

定义见 minimalist-ui（上游：minimalist-skill）

02 · Editorial Luxury

闻起来像：Kinfolk 杂志、精品酒店、高端地产、创意代理。

基底
暖奶油 #FDFBF7，叠一层很淡的噪点 / 胶片颗粒（约 3-5% 不透明度）
层级
巨大的高对比可变衬线标题，紧字距、紧行高
边缘
几乎没有：用分隔线和留白，不用盒子
招牌
鼠尾草绿或浓缩咖啡色点缀，元信息用小号宽字距大写，把留白当作产品本身
何时用
生活方式、地产、印刷相邻的品牌，凡是“考究”胜过“高效”的场景

定义见 high-end-visual-design §3.A.2（上游：soft-skill）

03 · Soft Structuralism

闻起来像：Apple 健康、高端消费养生、精修作品集。

基底
银灰 #F3F4F6 或纯白；冷、通透
层级
巨大的粗体怪诞体（800 字重、紧字距）
边缘
无边框。白色组件浮在极度弥散的环境阴影上；圆角 16-20px
招牌
带大数字的数据卡、胶囊 CTA、一切都在呼吸
何时用
消费级健康、个人指标、面向大众的亲和高端产品

定义见 high-end-visual-design §3.A.3（上游：soft-skill）

04 · Pristine Light

闻起来像：高质量印刷品、Stripe 的编辑页面、自信的文档。

基底
米白纸感 #FAF7F2，不玩纹理花招
层级
锐利的近黑无衬线，字号层级强烈；带编号的索引行
边缘
用发丝分隔线代替卡片；圆角约 0-2px；无阴影
招牌
读起来像排得很好的书页：编辑式自信，零装饰
何时用
内容优先的页面、方法论、宣言、想显得诚实的定价页

定义见 imagegen-frontend-web §2, Theme Paradigm 1

05 · Quiet Premium Neutral

闻起来像：Aesop、高端护肤、精品水疗、克制的奢侈品。

基底
骨白、沙、灰褐、石：#E9E4DB 家族，绝不用纯白
层级
细到常规字重、宽字距的大写标识、小字号
边缘
近直角（圆角 2-3px），CTA 用一条单色细边
招牌
刻意压低对比度；克制本身就是品牌
何时用
喊叫会掉价的高端货；注意低对比的无障碍代价，检查正文仍达标

定义见 imagegen-frontend-web §2, Theme Paradigm 4

暗色与大胆家族（第 05 课）

06 · Ethereal Glass

闻起来像：AI 创业公司、Linear、Vercel、Arc、暗色 SaaS。

基底
最深的 OLED 黑 #050505，暗色光球（翡翠绿、蓝）构成径向网格
层级
宽的几何怪诞体，近白色压在黑上
边缘
磨砂卡片：backdrop-filter: blur() + 1px white/14 边框 + 内高光（inset 0 1px 0 rgba(255,255,255,.2)）构成折射亮边；圆角 16px
招牌
卡片能看穿。为 prefers-reduced-transparency 提供纯色回退
何时用
AI 与开发者工具、高端科技、苹果相邻的产品发布

定义见 high-end-visual-design §3.A.1；玻璃亮边见 design-taste-frontend §5

07 · Deep Dark Mode

闻起来像：GitHub 暗色、Raycast、Vercel 仪表盘，多数产品暗色模式。

基底
石墨 / 锌灰 #0E0E11，刻意不用纯黑
层级
粗体怪诞体、锌白文字；一个克制、少量使用的强调色
边缘
实心卡片 #17171C 配 1px #26262E 边框；圆角 12px；无模糊、无磨砂
招牌
主力产品暗色模式。有理由时才用优雅的微光
何时用
仪表盘、开发者产品，任何需要平静、可读暗色界面的场景

定义见 imagegen-frontend-web §2, Theme Paradigm 2

08 · Dark Tech / Hacker

闻起来像：开发者工具、CLI、安全产品、终端优先的品牌。

基底
近黑、微微带绿 #0A0F0A
层级
纯等宽字体；霓虹绿 #4ADE80 强调色
边缘
终端窗口外壳、细的暗绿边框、方块化
招牌
终端元素：你敲的提示符、闪烁光标、命令输出。你在这里敲
何时用
开发者工具、CLI、安全，任何想要工程师可信度的场景

定义见 design-taste-frontend §2.B（dark tech / hacker）

09 · Tactical Telemetry

闻起来像：航空 HUD、交易终端、任务控制台、科幻界面。

基底
深黑 #06080A
层级
等宽字体、带微光的琥珀荧光 #FFB000；表格数字
边缘
ASCII 框架：角括号、准星刻度、虚线分隔
招牌
仪表读数：坐标、状态块、高密度数据。你在这里监看
何时用
数据密集的监控、运维，任何想要任务控制台质感的场景

定义见 industrial-brutalist-ui §2.2（第 06 课深挖）

10 · Bold Studio Solid

闻起来像：Gumroad、Linktree、大胆的 DTC 与创作者品牌。

基底
一整块满铺的高饱和色：宝蓝、牛血红、森绿、朱红或翡翠绿
层级
厚重怪诞体，压在色块上的锐利高对比近白文字
边缘
锐利，圆角 4px；只留一个亮的对比色（如黄色）给 CTA
招牌
唯一不走暗色的大胆家族。用一个下定决心的颜色表达自信
何时用
创作者工具、DTC、发布，任何品牌想显得响亮而笃定的场景

定义见 imagegen-frontend-web §2, Theme Paradigm 3

区分那些像的

PUM vs Pristine
PUM 是应用（外壳、kbd 按键、带边框的块、粉彩标签）。Pristine 是页面（分隔线、无容器、印刷质感）。
Pristine vs Luxury
都在纸色上。Luxury 以巨大衬线和留白领衔；Pristine 以锐利无衬线和清晰领衔。
Quiet vs 其余
背景变成石色或沙色、对比刻意压低：你就在 Quiet Neutral。
Glass vs Deep Dark
Glass 是磨砂（卡片能看穿）压在带光球的 OLED 黑上。Deep Dark 是实心石墨卡片，无模糊。
Hacker vs Telemetry
都是黑底霓虹等宽。Hacker 是你敲命令的终端；Telemetry 是你监看的仪表。
Studio vs 全部
只有 Studio Solid 彻底抛开黑与白，换一整块笃定的高饱和色。

两种 Brutalism（第 06 课）

一个家族，两种只共享一个红色的下定决心模式。每个项目选一种，让每一层都服从它；把两者混在一起，读起来就像穿了一身戏服。（Tactical Telemetry 就是你在 #09 见过的那个家族，这里是它作为 Brutalism 暗色模式的样子。）

11 · Swiss Industrial Print

闻起来像：1960 年代瑞士企业形象、机械蓝图、粗野主义编辑设计。

基底
未漂白文档纸 #F4F4F0 / #EAE8E3；碳黑墨 #0A0A0A
层级
宏观大字：厚重新怪诞体（Archivo Black、Neue Haas），clamp() 级字号，全大写，字距 -0.03 到 -0.06em，行高 0.85-0.95
边缘
可见的蓝图网格，1-2px 实线分隔满宽贯穿，圆角 0
招牌
被留白框住、溢出视口的超大数字；警示红 #E61919 作唯一强调色，用于结构分隔线和删除线
何时用
数据密集的编辑设计、作品集，任何想要「解密蓝图」式权威感的场景

定义见 industrial-brutalist-ui §2.1（上游：brutalist-skill）

12 · Tactical Telemetry

闻起来像：机密军用终端、大型主机、航空 HUD。

基底
已关闭的 CRT 黑 #0A0A0A / #121212，绝不用纯 #000
层级
微观小字：纯等宽字体（JetBrains Mono、IBM Plex Mono），全大写，字距 0.05-0.1em，固定小字号
边缘
ASCII 框架（[ ... ]、>>>），网格交点处的十字准星，1px 分隔线，圆角 0
招牌
扫描线叠加磷光 #EAEAEA 文字；警示红强调色；终端绿 #4AF626 只用在恰好一个元素上，绝不用于正文
何时用
高密度的监控与遥测、运维仪表盘，想要科幻感或任务控制台质感的场景

定义见 industrial-brutalist-ui §2.2（上游：brutalist-skill）
