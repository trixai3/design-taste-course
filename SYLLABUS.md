# Syllabus

Sixteen short lessons in four modules. The arc: **see** what generic looks like → **name** styles and
patterns → learn the **craft** per axis → **judge** which rule applies when. Every lesson is one
interactive HTML page with a single takeaway.

Status: ✅ live · ⏳ in progress.

## Module A — Seeing (calibrate the eye)
- ✅ **01 · The Tell** — spot and name the concrete signatures of AI-generated design.
- ✅ **02 · The Three Dials** — describe any page as Variance / Motion / Density (a triple like 7/6/4).
- ✅ **03 · The Design Read** — the audience picks the aesthetic; write a one-line read for any brief.

## Module B — Naming (style families & pattern vocabulary)
- ✅ **04 · Style Atlas I** — the light-mode families (minimalism, editorial luxury, soft structuralism…).
- ✅ **05 · Style Atlas II** — the dark & bold families (glassmorphism, dark tech, tactical telemetry…).
- ✅ **06 · The Two Brutalisms** — how one style splits into two committed modes you never mix.
- ✅ **07 · Pattern Glossary I** — hero, bento, masonry, composition anchors: layout names on sight.
- ✅ **08 · Pattern Glossary II** — motion vocabulary: sticky-stack, scrub, stagger, marquee, spring.

## Module C — Craft (best practice per axis)
- ✅ **09 · Typography** — fonts have voices; display vs body discipline; when a serif is earned.
- ✅ **10 · Color** — neutrals do the work, one accent gets the glory; the palettes to avoid.
- ⏳ **11 · Space & Surface** — why expensive sites feel expensive (whitespace, radius, shadow, grain).
- ⏳ **12 · Copy is Design** — the words that make a page read as generated.

## Module D — Judging (taste as context, then output)
- ⏳ **13 · Where the Rules Fight** — rules encode contexts, not truths; taste is knowing which you're in.
- ⏳ **14 · The Audit** — read an existing page like a designer and fix it in priority order.
- ⏳ **15 · Spot the Tell** — the final training game: find 15 planted tells from memory.
- ⏳ **16 · Capstone** — encode your own taste as a `DESIGN.md` that steers AI toward non-generic output.

---

Each lesson recommends a primary source and builds a reusable reference doc. The rules come from
[Leonxlnx/taste-skill](https://github.com/Leonxlnx/taste-skill) (MIT); this course turns them into
things you can see and practice.
